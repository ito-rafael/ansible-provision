- *LBiC*: The lab access control is done through a regular key. If you plan to use the physical space, kindly ask Prof. Von Zuben for a copy of the lab key.
#+LATEX: \vspace{-0.5cm}
* Accessing the computers
If you don't have your credentials yet, send an email\textsuperscript{\dag}  requesting it. To access hosts remotely, connect via VPN then SSH into the host. If you are locally connected to the lab's network, only an SSH connection needed.
- *VPN*: Connect to the lab VPN using your assigned credentials. A comprehensive setup guide was included in your credential email for reference.
- *SSH:* After establishing the VPN connection, log in to the PC and/or server:
  #+begin_src sh
  ssh your_username@ip_address
  #+end_src
#+LATEX: \vspace{-0.5cm}

* Mamba (Python environments) @@latex:\hfill\raisebox{-0.2\height}{\includegraphics[height=1.5em]{python.png}}\hspace{0.25cm}\raisebox{-0.2\height}{\includegraphics[height=1.5em]{mamba.png}}\hspace{0.25cm}\raisebox{-0.2\height}{\includegraphics[height=1.5em]{anaconda.png}}@@
*What is it?* Mamba is a package manager. It is a drop-in replacement for Conda, but rewritten in C++ to be drastically faster. It isolates your Python versions and libraries so different projects don't conflict.
#+LATEX: \vspace{1em}\noindent\rule{\linewidth}{0.4pt}\vspace{-1em}
** Create a new environment
Specify the Python version and any starting packages.
#+begin_src sh
mamba create -n <environment_name> python=3.10 pytorch -c pytorch
#+end_src
** List all environments
See all available environments on the server.
#+begin_src sh
mamba env list
#+end_src
** Activate the environment
Activate the environment every time you log in before running your code.
#+begin_src sh
mamba activate <environment_name>
#+end_src
** Install additional packages
Package installation can be done with =mamba= or =pip= (after environment activation).
#+begin_src sh
mamba install pandas
pip install scipy scikit-learn
#+end_src
** Deactivate the environment
Return to the default system environment.
#+begin_src sh
mamba deactivate
#+end_src
** Delete an environment
Clean up when your experiment is finished.
#+begin_src sh
mamba env remove -n <environment_name>
#+end_src
#+LATEX: \vspace{0em}\noindent\rule{\linewidth}{0.4pt}\vspace{0em}
** Example: train an AI model on MNIST dataset
#+ATTR_LATEX: :options basicstyle=\fontsize{8pt}{8.5pt}\ttfamily
#+begin_src sh
mamba create -n mnist_env python=3.10 -y
mamba activate mnist_env
pip install torch torchvision --upgrade --index-url https://download.pytorch.org/whl/cu128
wget -P ~/mnist_test https://raw.githubusercontent.com/pytorch/examples/main/mnist/main.py
cd mnist_test
python main.py --epochs 1
#+end_src

* Podman (containers) @@latex:\hfill\raisebox{-0.2\height}{\includegraphics[height=1.5em]{podman.png}}\hspace{0.25cm}\raisebox{-0.2\height}{\includegraphics[height=1.5em]{docker.png}}@@
# * Podman (containers) :ignore:
# +LATEX: \section*{Podman (containers) \hfill\raisebox{-0.2\height}{\includegraphics[height=1.5em]{podman.png}}\hspace{0.25cm}\raisebox{-0.2\height}{\includegraphics[height=1.5em]{docker.png}}}

*What is it?* Podman is a container engine. It is functionally identical to Docker (you can even use the alias "docker" to run the commands), but it is "daemonless" and runs securely without needing root privileges.
#+LATEX: \vspace{0em}\noindent\rule{\linewidth}{0.4pt}\vspace{0em}
** Run a container interactively
Download an image and run a command inside it.
#+begin_src sh
podman run --rm -it --device nvidia.com/gpu=all nvidia/pytorch:26.02-py3 bash
podman run --rm -it --device nvidia.com/gpu=all pytorch/pytorch:latest bash
#+end_src
** List running containers
Check what is currently running and get the container IDs.
#+begin_src sh
podman ps
#+end_src
** Enter inside a running container
#+begin_src sh
podman exec -it <container_id> bash
#+end_src
** List downloaded images
Check what container images are currently stored locally.
#+begin_src sh
podman image ls
#+end_src
** Stop a container
Stop a running container using its ID or name.
#+begin_src sh
podman stop <container_id>
#+end_src
** Remove a stopped container
#+begin_src sh
podman rm <container_id>
#+end_src
#+LATEX: \vspace{-0.25em}\noindent\rule{\linewidth}{0.4pt}\vspace{-0.25em}
** Example: run a Jupyter Notebook (with GPU)
#+LATEX: \vspace{0.5em}
Run a Jupyter container, map port =<xxxx>=, and keep it running in the background.
#+LATEX: \newline
If it asks for a token, run =podman logs <container_id>= to find the login URL.
#+begin_src sh
podman run -d --device nvidia.com/gpu=all -p <xxxx>:8888 jupyter/scipy-notebook
#+end_src
Run this on your local machine to map the server's Jupyter port to your own computer.
#+begin_src sh
ssh -L <xxxx>:localhost:<xxxx> -Nf user@server_ip
#+end_src
Then access =https://localhost:<xxxx>= on a web browser.
