\label{sec:org3594e00}
Prevent script termination on SSH disconnect by running \texttt{tmux} in the background.
\newline\newline
\noindent\begin{minipage}[t]{0.47\linewidth}
\subsection*{Start a new session}
\label{sec:org243c2f7}
\begin{lstlisting}[language=sh,numbers=none]

\end{lstlisting}

\label{sec:org3bf6b6a}
\begin{lstlisting}[language=sh,numbers=none,escapeinside={(*}{*)}]
(*Press \keys{\ctrl + b} $\longrightarrow$ \keys{d}*)
\end{lstlisting}
\subsection*{Reattach later}
\label{sec:orgc185131}
\begin{lstlisting}[language=sh,numbers=none]
tmux attach -t my_experiment
\end{lstlisting}
\subsection*{Split screen}
\label{sec:org1fd8fde}
\begin{lstlisting}[language=sh,numbers=none,escapeinside={(*}{*)}, lineskip=2pt, basicstyle=\ttfamily\footnotesize]

Horizontal:   (*\keys{\ctrl + b} $\longrightarrow$ \keys{"}*)
Switch panes: (*\keys{\ctrl + b} $\longrightarrow$ \keys{$\leftarrow$}\keys{$\uparrow$}\keys{$\downarrow$}\keys{$\rightarrow$}*)
\end{lstlisting}
\end{minipage}\hfill

\section*{Mamba (Python environments) \hfill\raisebox{-0.2\height}{\includegraphics[height=1.5em]{figure/python.png}}\hspace{0.25cm}\raisebox{-0.2\height}{\includegraphics[height=1.5em]{figure/mamba.png}}\hspace{0.25cm}\raisebox{-0.2\height}{\includegraphics[height=1.5em]{figure/anaconda.png}}}
\label{sec:org518e006}
\textbf{What is it?} Mamba is a package manager. It is a drop-in replacement for Conda, but rewritten in C++ to be drastically faster. It isolates your Python versions and libraries so different projects don't conflict.
\vspace{1em}\noindent\rule{\linewidth}{0.4pt}\vspace{-1em}
\subsection*{Create a new environment}
\label{sec:org21d962d}
Specify the Python version and any starting packages.
\begin{lstlisting}[language=sh,numbers=none]
mamba create -n <environment_name> python=3.10 pytorch -c pytorch
\end{lstlisting}
\subsection*{List all environments}
\label{sec:orgcfe8a73}
See all available environments on the server.
\begin{lstlisting}[language=sh,numbers=none]
mamba env list
\end{lstlisting}
\subsection*{Activate the environment}
\label{sec:org9143bc5}
Activate the environment every time you log in before running your code.
\begin{lstlisting}[language=sh,numbers=none]
mamba activate <environment_name>
\end{lstlisting}
\subsection*{Install additional packages}
\label{sec:orgd339ec6}
Package installation can be done with \texttt{mamba} or \texttt{pip} (after environment activation).
\begin{lstlisting}[language=sh,numbers=none]
mamba install pandas
pip install scipy scikit-learn
\end{lstlisting}
\subsection*{Deactivate the environment}
\label{sec:org909c98e}
Return to the default system environment.
\begin{lstlisting}[language=sh,numbers=none]
mamba deactivate
\end{lstlisting}
\subsection*{Delete an environment}
\label{sec:org4ce9e2c}
Clean up when your experiment is finished.
\begin{lstlisting}[language=sh,numbers=none]
mamba env remove -n <environment_name>
\end{lstlisting}
\vspace{0em}\noindent\rule{\linewidth}{0.4pt}\vspace{0em}
\subsection*{Example: train an AI model on MNIST dataset}
\label{sec:orge74f230}
\begin{lstlisting}[language=sh,numbers=none,basicstyle=\fontsize{8pt}{8.5pt}\ttfamily]
mamba create -n mnist_env python=3.10 -y
mamba activate mnist_env
pip install torch torchvision --upgrade --index-url https://download.pytorch.org/whl/cu128
wget -P ~/mnist_test https://raw.githubusercontent.com/pytorch/examples/main/mnist/main.py
cd mnist_test
python main.py --epochs 1
\end{lstlisting}

\section*{Podman (containers) \hfill\raisebox{-0.2\height}{\includegraphics[height=1.5em]{figure/podman.png}}\hspace{0.25cm}\raisebox{-0.2\height}{\includegraphics[height=1.5em]{figure/docker.png}}}
\label{sec:orge2ba046}
\textbf{What is it?} Podman is a container engine. It is functionally identical to Docker (you can even use the alias "docker" to run the commands), but it is "daemonless" and runs securely without needing root privileges.
\vspace{0em}\noindent\rule{\linewidth}{0.4pt}\vspace{0em}
\subsection*{Run a container interactively}
\label{sec:org53ebf52}
Download an image and run a command inside it.
\begin{lstlisting}[language=sh,numbers=none]
podman run --rm -it --device nvidia.com/gpu=all nvidia/pytorch:26.02-py3 bash
podman run --rm -it --device nvidia.com/gpu=all pytorch/pytorch:latest bash
\end{lstlisting}
\subsection*{List running containers}
\label{sec:org9f8485f}
Check what is currently running and get the container IDs.
\begin{lstlisting}[language=sh,numbers=none]
podman ps
\end{lstlisting}
\subsection*{Enter inside a running container}
\label{sec:org7077285}
\begin{lstlisting}[language=sh,numbers=none]
podman exec -it <container_id> bash
\end{lstlisting}
\subsection*{List downloaded images}
\label{sec:orgc90ba2d}
Check what container images are currently stored locally.
\begin{lstlisting}[language=sh,numbers=none]
podman image ls
\end{lstlisting}
\subsection*{Stop a container}
\label{sec:org817071f}
Stop a running container using its ID or name.
\begin{lstlisting}[language=sh,numbers=none]
podman stop <container_id>
\end{lstlisting}
\subsection*{Remove a stopped container}
\label{sec:org3973623}
\begin{lstlisting}[language=sh,numbers=none]
podman rm <container_id>
\end{lstlisting}
\vspace{-0.25em}\noindent\rule{\linewidth}{0.4pt}\vspace{-0.25em}
\subsection*{Example: run a Jupyter Notebook (with GPU)}
\label{sec:org3a970c1}
\vspace{0.5em}
Run a Jupyter container, map port \texttt{<xxxx>}, and keep it running in the background.
\newline
If it asks for a token, run \texttt{podman logs <container\_id>} to find the login URL.
\begin{lstlisting}[language=sh,numbers=none]
podman run -d --device nvidia.com/gpu=all -p <xxxx>:8888 jupyter/scipy-notebook
\end{lstlisting}
Run this on your local machine to map the server's Jupyter port to your own computer.
\begin{lstlisting}[language=sh,numbers=none]
ssh -L <xxxx>:localhost:<xxxx> -Nf username@server_ip
\end{lstlisting}
Then access \texttt{https://localhost:<xxxx>} on a web browser.
